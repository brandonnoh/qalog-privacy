QAlog — Privacy Policy

Last updated: April 6, 2026

1. What We Collect

QAlog collects the following data only when you actively submit a QA issue:

  • Email address — used to identify the submitter (via Google OAuth)
  • Screenshot — the area you select on the page
  • Page URL & title — the page where the issue was captured
  • DOM element info — tag name, CSS selector, XPath of a selected element (optional)
  • Browser environment — browser version, OS, viewport size, timezone, language
  • User-entered text — comment, repro steps, category, severity

2. Where Data Is Stored

All data is stored exclusively in your own Google Drive and Google Sheets, under your Google account. QAlog does not operate any servers and does not store any data outside of your Google account.

  • Screenshots → uploaded to a Google Drive folder owned by you
  • Issue records → appended to a Google Sheets spreadsheet owned by you

3. Third-Party Sharing

None. QAlog does not transmit data to any third-party servers, analytics services, or advertising networks. All communication occurs directly between your browser and Google APIs.

4. Google API Scopes

QAlog requests the following OAuth scopes:

  • userinfo.email — to identify the logged-in user
  • drive.file — to create and upload screenshots to a Drive folder created by the extension
  • drive.metadata.readonly — to detect the spreadsheet's parent folder for organizing screenshots
  • drive.appdata — to back up extension settings to a hidden app-specific folder (not visible to users)
  • spreadsheets — to append issue records to your spreadsheet

QAlog's use and transfer of information received from Google APIs adheres to the Google API Services User Data Policy, including the Limited Use requirements.

5. Data Retention

QAlog does not retain any data. Captured data exists only in your Google Drive and Sheets. You can delete it at any time through Google Drive.

6. Local Storage

QAlog uses chrome.storage.sync to store configuration (spreadsheet ID, folder ID, user preferences). This data stays within your Chrome profile and is not transmitted externally.

7. Children's Privacy

QAlog is not intended for children under 13 and does not knowingly collect personal information from children.

8. Changes

We may update this policy. Changes will be reflected on this page with an updated date.

9. Contact

If you have questions about this privacy policy, please open an issue on our GitHub repository.
